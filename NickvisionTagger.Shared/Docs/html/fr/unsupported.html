Fichiers non pris en charge

Cette page traité des fichiers musicaux qui ne sont actuellement pas pris en charge par Tagger.

Tagger utilise atldotnet comme backend pour lire et éditer les métadonnées.

Impossible d’appliquer les changements - Permission refusée

Il existe un problème connu de la version flatpak de l’application fonctionnant sur les systèmes basés sur Debian où il est impossible de sauvegarder des fichiers à cause d’une erreur « permission refusée ».

Consulter ce lien pour une solution : https://github.com/NickvisionApps/Tagger/issues/174

Veuillez confirmer que vous recevez une erreur « permission refusée » avant d’essayer cela.

Exécutez Tagger dans un terminal avec la commande flatpak run org.nickvision.tagger, puis essayez d’appliquer à nouveau les changements, vous trouverez le message d’erreur dans le terminal.

S’il ne s’egit pas d’une erreur « permission refusée », merci de signaler un nouveau problème.
